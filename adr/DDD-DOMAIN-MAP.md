## Bounded Context Details

### 1. Document Domain (Core)

**Purpose:** Model the lifecycle of a PDF document from upload to indexed state.

**Aggregate Root:** `IngestedDocument` (`app/ingestion.py`)

| Element | Type | Description |
|---|---|---|
| `IngestedDocument` | Aggregate Root | Carries all state for an indexed PDF (mode, pages, indices, IDs) |
| `PageChunk` | Value Object | Immutable (page_number: int, text: str) — unit of extracted content |
| `IngestedDocument.mode` | Domain Concept | `'pageindex'` or `'bm25'` — determines retrieval strategy |
| `doc_handle` | Identity | MD5 hash of file content — content-addressable identity |
| `cache.py` | Repository | Disk-based persistence (pickle) + in-memory registry |
| `ingestion.py` | Domain Service | `ingest_pdf()` — strategy-selects PageIndex vs BM25 pipeline |

**Invariants:**
- `mode` is immutable after ingestion
- `doc_handle = MD5(file_content)` — same file always maps to same handle
- `pageindex_client` excluded from pickle; reconstructed from env on load

**Ubiquitous Language:** ingest, doc_handle, page_chunk, cache hit, mode

---

### 2. Conversation Domain (Core)

**Purpose:** Manage user session state: message history and active document selection.

**Aggregate Root:** Session (implicit dict in `api/session.py`, identified by `session_id` UUID)

| Element | Type | Description |
|---|---|---|
| Session | Aggregate Root | `{history: List[Message], active_doc_id: str\|None, last_active: float}` |
| Message | Value Object | `{role: 'user'\|'assistant', content: str}` |
| `session_id` | Identity | UUID v4 generated by the browser frontend |
| SESSION_TTL | Domain Rule | 2-hour inactivity expiry |
| `active_doc_id` | Association | FK → Document domain via `doc_handle` |

**Invariants:**
- Session expires after `SESSION_TTL_SECONDS = 7200` of inactivity
- `active_doc_id` may be null (general chat mode)
- History grows unbounded within a session (no trim)

**Ubiquitous Language:** session, chat history, active document, select/deselect

---

### 3. Retrieval Domain (Supporting)

**Purpose:** Given a query and an `IngestedDocument`, return ranked relevant page chunks.

**Value Object:** `RetrievedChunk` (`app/retrieval.py`)

| Element | Type | Description |
|---|---|---|
| `RetrievedChunk` | Value Object | `{page_number, text, score, source}` |
| `retrieve()` | Domain Service | Strategy dispatch: delegates to PageIndex or BM25 |
| `_retrieve_pageindex()` | Strategy | Cloud retrieval + local BM25 for source pages |
| `_retrieve_bm25()` | Strategy | Local BM25 scoring over extracted pages |
| `source='pageindex_answer'` | Domain Concept | Synthetic chunk carrying PageIndex's pre-generated answer |

**Invariants:**
- `top_k` default = 5 chunks returned
- `source='pageindex_answer'` chunk inserted at index 0; `generate_node` short-circuits when it sees this

**Ubiquitous Language:** chunk, retrieve, top-k, relevance score, source

---

### 4. Orchestration Domain (Supporting)

**Purpose:** Classify queries and route them through the correct processing pipeline.

**Aggregate:** `ChatState` TypedDict + compiled `StateGraph` singleton

| Element | Type | Description |
|---|---|---|
| `ChatState` | Workflow State | Shared state passed between all graph nodes |
| `build_graph()` | Factory | Constructs and compiles the LangGraph `StateGraph` |
| `get_graph()` | Singleton | Process-level compiled graph instance |
| `classifier_node` | Policy | LLM-based 3-way classifier: general / document / agent |
| `run_chat_pipeline()` | Application Service | Entry point — sets initial state, invokes graph, returns result |
| ReAct loop | Process | `agent_node ↔ tool_node`, max 5 iterations |

**Routes:**
- `general` → `general_node` → LLM with chat history
- `document` → `retrieve_node → generate_node` → RAG
- `agent` → `agent_node ↔ tool_node → agent_final_node` → ReAct

**Ubiquitous Language:** route, classify, pipeline, iteration, tool call, state

---

### 5. LLM/AI Domain (Generic)

**Purpose:** Provide a consistent interface to external AI models and agent tools.

| Element | Type | Description |
|---|---|---|
| `get_llm()` | Factory | Creates `ChatOpenAI` pointing at OpenRouter |
| `OPENROUTER_BASE_URL` | Config | `https://openrouter.ai/api/v1` |
| `tavily_search` | Tool | Web search via Tavily API (opt-in) |
| `browser_navigate` | Tool | Playwright headless Chromium page reader |
| `browser_get_text` | Tool | CSS-selector-scoped page text extraction |
| `browser_screenshot` | Tool | Page text preview (screenshots not returned) |

**Ubiquitous Language:** model, tool, tool call, temperature, prompt

---

### 6. Observability Domain (Generic)

**Purpose:** Capture execution traces for debugging and performance analysis.

Currently embedded inside `app/graph.py` (`_is_langsmith_configured()`, `run_chat_pipeline()` tracing block). Not cleanly separated.

| Element | Type | Description |
|---|---|---|
| LangSmith client | Adapter | Per-request `ls.Client` instance |
| `tracing_context()` | Decorator | Wraps `graph.invoke()` with project-scoped trace |
| `client.flush()` | Side effect | Synchronous trace submission before response |

**Ubiquitous Language:** trace, span, project, flush

## Context Map (Relationships)

```
Conversation Domain
    ──(uses)──► Document Domain      [session.active_doc_id → doc_handle]
    ──(uses)──► Orchestration Domain [chat.py calls run_chat_pipeline()]

Orchestration Domain
    ──(uses)──► Document Domain      [active_doc passed in ChatState]
    ──(uses)──► Retrieval Domain     [retrieve_node calls retrieve()]
    ──(uses)──► LLM/AI Domain        [all nodes call get_llm(); agent uses tools]
    ──(uses)──► Observability Domain [run_chat_pipeline wraps graph.invoke()]

Retrieval Domain
    ──(uses)──► Document Domain      [reads IngestedDocument.mode, pages, etc.]

Document Domain
    ──(uses)──► LLM/AI Domain        [PageIndex cloud client in ingestion.py]
```

## Architectural Observations

1. **Observability is not a proper bounded context** — LangSmith logic is scattered inside `run_chat_pipeline()`. Should be extracted to an `app/observability.py` module.

2. **Session domain violates SRP** — `api/session.py` manages both session lifecycle AND the global document registry. These are separate concerns.

3. **No Application Service layer** — `api/routes/chat.py` directly calls `run_chat_pipeline()` and `sess.*` functions. A thin application service (`ChatApplicationService`) would clarify the boundary.

4. **RetrievedChunk uses a magic `source` string** (`'pageindex_answer'`) as a signal between Retrieval and Orchestration domains — this is an implicit coupling. A proper domain event or discriminated union would be cleaner.
